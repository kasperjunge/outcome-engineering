# I need collaborators to see the same product direction

When I work with a co-founder, collaborator, or small team, I need them to see the same product graph I am using without needing my local repo setup.

The graph can clarify vision, strategy, outcomes, and active work for me, but it creates less alignment if others cannot casually open it before or during product conversations. I need shared read access that makes the graph easy to revisit together while preserving the repo as the source of truth.

## Evidence

- Hypothesis: "I want my co-founders to see the same product graph I am using, without needing my local setup."
- Hypothesis: "I want to quickly revisit our product direction in a browser when we discuss what matters next."
- Hypothesis: Shared read access may be enough before shared editing is necessary.

## Known / Unknown

- Known: The existing graph UI is local and requires running `oe serve`.
- Known: The current strategy says the graph should be visually usable and help users stay aligned with the vision.
- Known: Shared human visibility matters most for small teams, founder pairs, and collaborators.
- Unknown: How often users will return to a read-only hosted graph.
- Unknown: Whether read-only visibility creates enough alignment value before collaboration and editing features are added.
