# Agent Entry Points

Start from the user's real material. Grow the graph only where it improves judgment or traceability.

## Challenge Assumptions

Use when the user has messy product thinking and wants to avoid building too long without validation.

Output: hidden assumptions, the riskiest assumption, suggested graph edits, and the next learning step.

## Craft Vision And Strategy

Use when the user needs clearer vision, strategy, outcomes, focus, or positioning.

Output: concise vision/strategy text, strategic choices, blocking questions, and suggested graph edits.

## Challenge Product Intent

Use when the user has a PRD, spec, issue, implementation plan, code change, or acceptance test.

Output: intended outcome, trace strength, evidence gaps, risky assumptions, and whether the work looks high leverage.

## Rule

Agents propose graph changes. Humans approve product truth.
