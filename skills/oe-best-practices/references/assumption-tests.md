# Assumption Tests

An assumption test is a small credible way to learn whether one risky belief behind a solution is true.

Solutions reveal assumptions. Assumption tests create evidence for whether those assumptions should still be believed.

Use these five rules when writing or reviewing assumption tests:

1. Test one risky assumption.
   Do not test the whole solution at once.

2. State the assumption clearly.
   Write the belief that must be true for the solution to work.

3. Choose the smallest credible test.
   Reduce uncertainty without overbuilding. The test can be an interview, prototype, concierge test, landing page, manual workflow, usability test, smoke test, data analysis, technical spike, or any other creative way to get credible evidence.

4. Define evidence before running it.
   Say what result would increase or decrease confidence.

5. Connect learning back to the solution.
   The value is the decision it enables: continue, change, stop, or learn more.
