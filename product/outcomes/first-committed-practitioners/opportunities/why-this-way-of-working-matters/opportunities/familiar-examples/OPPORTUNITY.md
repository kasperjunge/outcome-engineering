# Solo builders need familiar examples

Solo software product builders need to see Outcome Engineering applied to products they already understand. Familiar examples make the framework concrete, credible, and easier to imagine using on their own product.

## Evidence

- Hypothesis: Public product graph examples of known products could show how vision, strategy, outcomes, opportunities, solutions, assumptions, PRDs, and delivery connect in practice.
- Hypothesis: Recognizable examples reduce the cognitive load of learning a new framework because the product context is already familiar.

## Known / Unknown

- Known: New users are unlikely to understand the framework from taxonomy alone.
- Known: Examples can teach the framework without requiring the user to start by modeling their own product.
- Unknown: Which example products create the most excitement and least confusion.
- Unknown: Whether examples should be accurate strategic analysis, illustrative teaching artifacts, or community-editable graph case studies.
