# I need to see how my strategy compounds

The first user and strategy-minded early builders need a way to understand how the parts of their strategy reinforce each other over time. They may have a vision, a current strategic bet, outcomes, opportunities, and solution ideas, but still lack a clear picture of the motion that creates momentum.

Without that causal picture, strategy can feel like a list of priorities. Builders may choose work because it seems useful in isolation, not because it strengthens the system they are trying to create.

This matters because Outcome Engineering is meant to help builders focus on the highest-leverage next step. If the compounding logic is invisible, the builder can miss which part of the system is weakest, which work has strategic leverage, and which activities are merely busy.

## Evidence

- Hypothesis: The first user already thinks in flywheels and needs a structured place to capture and critique that thinking inside the product graph workflow.
- Hypothesis: Some later builders may also think in flywheels, but the core graph should remain useful without this concept.
- Hypothesis: A visual causal loop could help builders see weak links in strategy faster than prose alone.
- Hypothesis: The flywheel should sit beside the normal trace graph because it explains strategic momentum, not delivery traceability.

## Known / Unknown

- Known: Outcome Engineering already keeps vision, strategy, outcomes, opportunities, solutions, assumptions, PRDs, and delivery work connected.
- Known: The current product strategy says users should be able to start with whatever is on their mind and leave with clearer product judgment.
- Known: A flywheel is a graph of reinforcing causes, but it is not the same shape as the outcome -> opportunity -> solution trace chain.
- Unknown: Whether this need is common beyond the first user or mostly valuable to strategy-minded power users first.
- Unknown: Whether a flywheel view should be introduced during strategy work, outcome planning, or only as an optional advanced surface.
- Unknown: What level of structure helps without turning the flywheel into heavy methodology.
