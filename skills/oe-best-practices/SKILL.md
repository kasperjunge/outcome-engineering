# OE Best Practices

Use this skill to keep Outcome Engineering product graph content semantically clean.

## Workflow

1. Identify the node type being created, edited, reviewed, or classified.
2. Read the matching reference file before writing or judging the node.
3. Apply the reference as a contract: what belongs, what does not, title shape, body shape, and smells.
4. If the user's input mixes node types, separate the concepts before writing.
