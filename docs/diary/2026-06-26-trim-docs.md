**Author:** main

### What I did

I created the `/.worktrees/trim-docs` worktree on branch `trim-docs`. I reduced `/README.md` to a short product description, the core CLI commands, and links to only the docs that still matter. I trimmed `/docs/agent-entry-points.md` to the three strategic agent entry points and the approval boundary. I removed `/docs/framework.md`, `/docs/example-structure.md`, `/docs/graph.md`, and `/docs/glossary.md` because they were reference or explanatory material that did not directly support the current vision and strategy.

### Why

The current strategy is agent-native: challenge assumptions, craft vision and strategy, and challenge product intent. The retained docs support that shape directly; the removed docs duplicated earlier explanations or documented generic graph concepts.

### What worked

`uv run oe validate product` passed. `uv run pytest` passed with 14 tests. A stale-link search found only historical diary references to the deleted docs, which should remain unchanged.

### What didn't work

Nothing failed during this cleanup.

### What I learned

The repository can explain the product adequately with `/product/VISION.md`, `/product/STRATEGY.md`, `/docs/agent-entry-points.md`, and a short README. The bundled skills and CLI help carry the operational detail.

### What was tricky

The main judgment call was whether graph and glossary docs counted as signal. Under the stricter criterion of supporting current vision and strategy directly, they were reference material rather than strategic material.
